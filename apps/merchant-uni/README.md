# 商户端 uni-app

首期覆盖经营概况、订单接单/拒单/出餐、商品管理、店铺设置、钱包和消息。

当前已落地：

- 经营概况页：展示商户接单状态、待接单、备餐中、待骑手订单计数。
- 订单处理页：支持商户订单列表、接单、出餐，并调用 BFF/API。
- 商品管理页：支持商品列表、新增样例、上架、售罄、下架，并调用 BFF/API。
- 团购核销页：支持相机扫码和手输二维码内容，调用 BFF/API 核销用户团购券。
- 资质资料页：展示接单门槛、主体资质缺失、商户保证金、员工健康证和补充资料，并调用 BFF/API 提交营业执照、健康证、员工信息和门头/后厨等资料。
- API 客户端：默认连接 `http://127.0.0.1:25500`，支持商户邀请注册设置密码、商户账号密码登录并保存 `merchantToken`；匿名注册/登录请求不会携带开发 token。
- 开发兜底：未登录时默认开发 token 为 `merchant:merchant_1`，便于本地预览既有经营页面。
- 本地预览兜底数据：后端未启动时仍可看页面结构。

商家账号与店铺关系：

- 商家账号管理主体、权限、资质、合同、结算和钱包。
- 商家端只能通过管理员邀请链接注册，不能自行注册；邀请注册时必须设置 8-72 字节密码，后续通过 `account_id + password` 登录。
- 店铺负责展示页、商品、团购套餐、评价、公告和服务能力。
- 同一个店铺可以同时开通外卖和团购。
- 药房/医务室账号或店铺需要资质审核后才能开通买药能力。
- 营业执照和健康证必须上传文件并填写失效日期。
- 资质过期或未审核通过时，店铺暂时关闭，商户端打开后弹窗提示补资料。
- 商户端需要维护员工信息、员工健康证、门头照/后厨照/许可证等补充资料。
- 商户缴纳 50 元保证金后才可接单，商户不支持微信免押。
- 商户可自主维护店铺详情页：头图、logo、公告、优惠、评价展示、资质展示、外卖商品和团购套餐。
- 商品/菜品管理必须支持图片、描述、配料表、价格、库存、上下架和售罄状态。
- 团购商户端必须提供二维码扫描器，扫描用户团购券二维码后完成核销。
- 团购商品下架、售罄或库存为 0 时，未核销券进入自动退款流程，默认退到用户平台余额。
- 商户可创建商户群，并配置优惠券必须进群后才能领取或使用。
- 商户可自行发券，商户券优惠成本由商户承担。
- 平台活动券如果要求商户承担成本，商户端必须弹窗展示活动规则，商户点击同意后才参与。
- 商户券适用范围可以是本店，平台活动券适用范围可以是参与活动的全部商家。
- 商户可通过群聊或私聊给用户、骑手发送余额红包，支持普通定额红包和拼手气红包。
