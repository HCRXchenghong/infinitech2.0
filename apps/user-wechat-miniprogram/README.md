# 用户端原生微信小程序

首期定位：外卖和团购主展示，首页模块化扩展买药、快递/跑腿、后续交友。

当前首批页面：

- `pages/index/index`：首页模块、推荐卡片、圈子入口。
- `pages/shop/list/index`：商家列表，外卖/团购/买药/跑腿入口共用。
- `pages/shop/detail/index`：美团式店铺详情骨架，包含外卖商品和团购券，已接入团购套餐列表和下单入口。
- `pages/cart/index`：购物车和金额拆分。
- `pages/order/confirm/index`：确认订单、地址、备注、餐具和支付入口。
- `pages/order/list/index`：订单列表。
- `pages/order/detail/index`：订单详情和订单事件流。
- `pages/wallet/payment-password/index`：余额支付密码设置。
- `pages/address/list/index`：收货地址列表。
- `pages/circle/index`、`pages/meal-match/index`：圈子和找饭搭入口。
- `utils/api.ts`：小程序 API 客户端，支持微信登录、店铺、商品、团购、购物车、结算、订单、支付密码和微信支付预下单接口接入。
- 品牌资源已复制到 `assets/brand/`，供小程序本地引用。
- 预览图：`docs/product/user-miniprogram-preview.png`。

约束：

- 原生微信小程序，不使用旧版用户端 uni-app 作为主线。
- 只访问 BFF 或公开 API。
- 主色固定 `#009bf5`，logo 来源 `assets/brand/logo.svg`。
- 商家店铺页统一展示外卖商品、团购套餐、评价、公告、商家资质和联系入口。
- 团购订单到店扫码验券；快递/跑腿进入骑手履约流程。
- 首页增加圈子入口，圈子首期是后台可控的小微墙，不整仓嵌入 InfiniLink。
- 找饭搭归属圈子能力，使用前必须完善性别、签署身份真实性承诺和平台免责承诺，并完成性格与饮食习惯问卷。
- 首页推荐商品、店铺、团购、优惠券和圈子卡片都从后台配置读取。
- 旧版已有能力必须保留：搜索、今日推荐、商品详情、购物车、地址、备注、餐具数量、支付结果、订单追踪、评价、售后、收藏、我的评价、红包优惠、钱包充值/提现/账单、积分商城、会员中心、反馈合作、公益。
- 余额支付前必须设置余额支付密码，密码未设置或锁定时不允许余额支付。
- 退款默认退回平台余额，后续以后台退款策略为准。
- 消息页包含私聊、官方群、商户群；用户注册后自动加入官方群，默认不通知。
- 用户可在群聊或私聊中用余额发送普通红包或拼手气红包。
- 指定优惠券可配置为必须加入商户群后才能领取或使用。
